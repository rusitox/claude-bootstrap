Plan the implementation for: $ARGUMENTS

1. **Understand the requirement**: Clarify goal, scope, constraints.
2. **Explore the codebase**: Identify relevant templates, scripts, and existing patterns.
3. **Identify dependencies**: Which templates need updating, new placeholders required, script changes needed.
4. **Create a phased plan** and save to `specs/plan-[feature-name].md`
5. Present for review before implementation.
6. Never write code during planning — only analyze and plan.
